## Voxel Grid

VoxCSS works on a **3D coordinate system** defined by three axes: **x** (columns), **y** (rows), and **z** (depth/layers). Every voxel occupies a cell in this grid.

Internally, the engine stacks flat CSS grid layers along the Z axis. Each layer is a 2D grid positioned with a CSS `translateZ` transform. The result is a fully 3D structure built entirely from nested HTML elements — no canvas or WebGL involved.

Coordinates are zero-indexed integers. `z: 0` is the ground layer, and higher Z values move upward.

## Voxel Data Model

Each voxel is a plain object. The only required fields are grid coordinates:

- **`x`, `y`, `z`** — position in the grid (required).
- **`x2`, `y2`, `z2`** — extend the voxel into a rectangular area spanning from `(x, y, z)` to `(x2, y2, z2)`. Useful for floors, walls, or any multi-cell block.
- **`color`** — any CSS color value.
- **`texture`** — apply an image or pattern to the voxel faces.
- **`shape`** — the geometry to render (see Shapes below).
- **`rot`** — rotation of the shape, snapped to 90-degree increments.

Because voxels are plain objects, you can generate them from loops, load them from JSON, or compute them from any data source.

## Rendering Pipeline

VoxCSS is structured in three layers:

1. **Core** — Pure math and data. Computes merged geometries, face visibility, and grid layout. No DOM dependency — this layer can run anywhere.
2. **HTML Renderer** — Takes the core output and produces real DOM elements. It creates the CSS grid layers, positions cuboids with transforms, and applies colors and textures.
3. **Framework Wrappers** — Thin bindings for React and Vue that expose the engine as declarative components (`VoxCamera`, `VoxScene`). These wrap the HTML renderer and handle reactivity, lifecycle, and prop updates for you.

The vanilla JS `renderScene` function uses the HTML renderer directly, skipping the framework layer entirely.

## Merge Modes

When many adjacent voxels share a color, VoxCSS can **merge** them into fewer, larger HTML elements. Toggle merge 3d below to see the difference — keep an eye on how the rendered DOM reacts in DevTools.

<VoxDemo
  id="concept-merge"
  model="apple"
  controls='["zoom","mergeVoxels","showFloor"]'
  defaults='{"zoom":0.5,"interactive":true,"showFloor":false,"mergeVoxels":"3d"}'
/>

- **`false`** — No merging. Every voxel is its own element. Best for scenes where each voxel needs individual interaction.
- **`"2d"`** — Merges adjacent same-color voxels within each Z layer.
- **`"3d"`** — Slice-based rendering. Fewest elements, best raw performance, no per-voxel addressing (cubes only).

See the **[Performance guide](/guides/performance)** for a live FPS/DOM stats demo and guidance on choosing a mode.
