---
title: Headless API
description: Vanilla JavaScript API for loading meshes and rendering polygon scenes without a framework.
---

The headless API (`@polycss/core`) gives you direct access to polycss parsers and utilities without React, Vue, or any framework.

## `parseObj(text, options?)`

Parses an OBJ file text string into a polygon array.

```ts
import { parseObj } from "@polycss/core";

const result = parseObj(objText, {
  targetSize: 60,
  defaultColor: "#cccccc",
  mtlText: mtlFileText,           // optional .mtl file contents
});
// result.polygons — Polygon[]
// result.dispose() — revoke any blob URLs
```

**Options:** See [`ObjParseOptions`](/api/types/#objparseoptions).

---

## `parseGltf(buffer, options?)`

Parses a GLB or glTF file into a polygon array. Accepts `ArrayBuffer`.

```ts
import { parseGltf } from "@polycss/core";

const buf = await fetch("/model.glb").then(r => r.arrayBuffer());
const result = parseGltf(buf, {
  targetSize: 60,
  baseUrl: new URL("/model.glb", location.href).href,
});
// result.polygons — Polygon[]
// result.objectUrls — blob: URLs for embedded textures
// result.dispose() — revoke all objectUrls
```

**Options:** See [`GltfParseOptions`](/api/types/#gltfparseoptions).

---

## `parseMtl(text)`

Parses an MTL file text string into a material map.

```ts
import { parseMtl } from "@polycss/core";

const { colors, textures } = parseMtl(mtlText);
// colors — Record<materialName, cssColor>
// textures — Record<materialName, imagePath>
```

---

## `loadMesh(url, options?)`

High-level convenience function: fetches a URL, detects OBJ vs GLB/glTF by extension, parses it, and returns a `ParseResult`.

```ts
import { loadMesh } from "@polycss/core";

const result = await loadMesh("/cottage.glb", { targetSize: 60 });
// result.polygons — Polygon[]
// result.dispose() — call on cleanup
```

---

## `normalizePolygons(polygons)`

Computes bounding boxes and fills in any missing derived fields.

```ts
import { normalizePolygons } from "@polycss/core";

const normalized = normalizePolygons(rawPolygons);
```

---

## `createIsometricCamera(options)`

Creates an isometric-style camera state object (useful for the imperative rendering API).

```ts
import { createIsometricCamera } from "@polycss/core";

const cam = createIsometricCamera({
  rotX: 65,
  rotY: 45,
  perspective: 1000,
});
```

---

## Custom elements (vanilla)

Register the `<poly-scene>` and `<poly-mesh>` custom elements by importing the side-effect entry point:

```html
<script type="module" src="https://esm.sh/polycss/elements"></script>

<poly-scene perspective="1000" rot-x="65" rot-y="45">
  <poly-mesh src="/cottage.glb"></poly-mesh>
</poly-scene>
```

## Package Exports

| Import path         | Contents                                    |
| ------------------- | ------------------------------------------- |
| `@polycss/react`    | React components: `PolyScene`, `PolyMesh`, `Poly`, `PolyCamera`, hooks |
| `@polycss/vue`      | Vue components: `PolyScene`, `PolyMesh`, `Poly`, `PolyCamera`         |
| `polycss`           | Vanilla imperative API + custom elements    |
| `polycss/elements`  | Side-effect: registers `<poly-scene>` etc.  |
| `@polycss/core`     | Pure parsers / math, zero DOM — `parseObj`, `parseGltf`, `loadMesh`, types |
