The camera component wraps a scene and controls how it is viewed — perspective depth, rotation angles, and zoom. It's available as a React / Vue component (`<PolyCamera>`); vanilla users should set the same attributes (`rot-x`, `rot-y`, `perspective`, `zoom`) directly on `<poly-scene>`.

For most use cases, pass camera attributes directly to the scene element / `PolyScene` component instead. The camera wrapper is useful when you need imperative camera control (animating the camera from code, syncing it to external state, or splitting camera state and scene layout into separate React/Vue components).

For pointer drag, wheel zoom, and autorotate, drop a `<PolyControls>` child inside the scene — see **[PolyControls](/components/poly-controls)**.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `zoom` | `number` | `1` | Scales the scene. `1` is the natural size; values above 1 zoom in, below 1 zoom out. |
| `rotX` | `number` | `65` | Rotation around the X axis in degrees. |
| `rotY` | `number` | `45` | Rotation around the Y axis in degrees (0–360). |
| `perspective` | `number \| false` | `8000` | CSS perspective depth in pixels. Pass `false` for orthographic projection. |

`<PolyCamera>` is a state holder — it owns the camera transform and exposes it to descendants via context. For pointer drag, wheel zoom, and autorotate, mount [`<PolyControls>`](/components/poly-controls) inside `<PolyCamera>`. This mirrors Three.js's split between `PerspectiveCamera` (state) and `OrbitControls` (input).

## Usage

### Basic Camera

Set an initial angle and perspective.

<Tabs syncKey="fw">
<TabItem label="Vanilla JS">
```html
<script type="module" src="https://esm.sh/@layoutit/polycss/elements"></script>

<!-- Vanilla scenes set camera attributes directly on <poly-scene>. -->
<poly-scene zoom="0.8" rot-x="70" rot-y="30">
  <poly-mesh src="/cottage.glb"></poly-mesh>
</poly-scene>
```
</TabItem>
<TabItem label="React">
```tsx
import { PolyCamera, PolyScene, PolyMesh } from "@layoutit/polycss-react";

export function App() {
  return (
    <PolyCamera zoom={0.8} rotX={70} rotY={30}>
      <PolyScene>
        <PolyMesh src="/cottage.glb" />
      </PolyScene>
    </PolyCamera>
  );
}
```
</TabItem>
<TabItem label="Vue">
```vue
<template>
  <PolyCamera :zoom="0.8" :rot-x="70" :rot-y="30">
    <PolyScene>
      <PolyMesh src="/cottage.glb" />
    </PolyScene>
  </PolyCamera>

import { PolyCamera, PolyScene, PolyMesh } from "@layoutit/polycss-vue";

```

### Interactive Camera with Auto-rotation (recommended)

Drop a `<PolyControls>` child for drag, wheel, and `dt`-clamped autorotate. Works the same across vanilla, React, and Vue.

```html
<!-- Vanilla -->
<poly-scene rot-x="65" rot-y="45">
  <poly-controls drag wheel animate-speed="0.5"></poly-controls>
  <poly-mesh src="/cottage.glb"></poly-mesh>
</poly-scene>
```

```tsx
// React
<PolyCamera rotX={65} rotY={45}>
  <PolyScene>
    <PolyControls drag wheel animate={{ axis: "y", speed: 0.5 }} />
    <PolyMesh src="/cottage.glb" />
  </PolyScene>
</PolyCamera>
```

Full options reference: **[PolyControls](/components/poly-controls)**.

## `useCamera` hook (React)

`useCamera` is used internally by `<PolyCamera>`. For most use cases, pass camera props directly to `<PolyScene>` — it accepts all of `PolyCamera`'s state-shaping props. Use `<PolyCamera>` as a wrapper when you need to separate camera state from scene layout.
