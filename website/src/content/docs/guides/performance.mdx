VoxCSS renders everything as real DOM elements — every face of every voxel is an HTML element with CSS transforms. That gives you DevTools inspection, DOM events, and CSS styling on every voxel, but it means **performance scales with element count**. Understanding how to manage that count is key to building smooth scenes.

## Face culling

The engine automatically hides faces that aren't visible. Two types of culling happen out of the box:

- **Camera culling** — faces pointing away from the current camera rotation are hidden via CSS.
- **Neighbor occlusion** — when two voxels share a face (e.g., a voxel at `(1,1,0)` sits next to one at `(2,1,0)`), the shared faces between them are removed. They'd never be seen, so there's no reason to render them.

This culling is automatic and requires no configuration. It significantly reduces the number of rendered elements in dense scenes.

## Merge modes

The `mergeVoxels` prop on `<VoxScene>` controls how aggressively the engine consolidates adjacent same-color voxels into larger elements.

### No merging (default)

Each voxel is its own DOM element. Full per-voxel control — every voxel can be individually targeted with CSS selectors, event listeners, or animations.

**Best for:** small scenes, interactive voxels, scenes where each voxel needs distinct behavior.

### 2D merging — `"2d"`

Merges adjacent same-color voxels **within each Z layer**. The engine finds rectangular regions of identical voxels on each flat layer and combines them into single, larger elements.

**Best for:** medium scenes with large flat areas of uniform color — floors, walls, terrain surfaces.

### 3D merging — `"3d"`

Volumetric merge across **all three axes**. The engine switches to a slice-based rendering approach, producing the fewest possible elements. Adjacent same-color voxels are combined into large cuboid regions regardless of layer boundaries.

**Best for:** large, dense models where raw performance matters more than per-voxel control — imported `.vox` files, detailed scenes, anything visibly laggy at the default setting.

## Setting merge mode

`mergeVoxels` is a prop on `<VoxScene>`. See [VoxScene reference](/components/vox-scene#props) for the full prop table.

- **React:** `<VoxScene voxels={voxels} mergeVoxels="2d" />`
- **Vue:** `<VoxScene :voxels="voxels" merge-voxels="2d" />`
- **Vanilla JS:** `scene: { voxels, mergeVoxels: "2d" }`

## Automatic 3D path

If any voxel in your scene uses `z2` (spanning multiple Z layers with area spans like `{ x: 0, y: 0, z: 0, z2: 3 }`), the engine automatically switches to the 3D rendering path regardless of your `mergeVoxels` setting. The slice-based renderer is required to handle multi-layer spans correctly.

## Choosing a merge mode

| Mode | DOM nodes | Per-voxel control | Best use case |
|------|-----------|-------------------|---------------|
| `false` | Most | Full | Small scenes, interactive voxels |
| `"2d"` | Fewer | Limited | Medium scenes, flat color regions |
| `"3d"` | Fewest | None | Large dense models, imported `.vox` files |

Start with the default (no merging) and switch to `"2d"` or `"3d"` if you notice rendering lag. In many cases, the automatic face culling alone is enough to keep scenes performant.
