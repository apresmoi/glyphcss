## Basic usage

Set the `texture` property to an image URL. The image is applied as a CSS background on each face of the voxel.

<Tabs syncKey="fw">
<TabItem label="React">
```tsx
import { VoxCamera, VoxScene } from "@layoutit/voxcss/react";

const voxels = [
  { x: 1, y: 1, z: 0, texture: "/grass.png" },
  { x: 2, y: 1, z: 0, texture: "/grass.png" },
  { x: 1, y: 2, z: 0, texture: "/grass.png" },
];

export function App() {
  return (
    <VoxCamera interactive>
      <VoxScene voxels={voxels} />
    </VoxCamera>
  );
}
```

## Combining texture and color

You can set both `texture` and `color` on the same voxel. The color acts as a **fallback** while the image loads (so voxels don't flash white) and as a **tint** that blends with the texture.

```ts
{ x: 1, y: 1, z: 0, color: "#8B4513", texture: "/wood.png" }
```

## Texture sources

The `texture` value is any valid CSS image URL:

- **Relative paths** — files in your project's public directory (e.g., `/grass.png`)
- **Absolute URLs** — hosted images (e.g., `https://example.com/texture.png`)
- **Data URIs** — inline base64-encoded images

## Tips

- Use **small, tileable images** (e.g., 16×16 or 32×32 pixels) for the best look at voxel scale.
- Textures render via CSS `background-image`, so they benefit from browser caching automatically.
- For scenes with many textured voxels, reuse the same image URL — the browser only fetches it once.
