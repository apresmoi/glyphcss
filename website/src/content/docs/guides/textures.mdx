polycss loads 3D mesh files (OBJ, GLB, glTF) and renders them as DOM elements. UV textures are extracted from the file and applied to each polygon via a one-time off-DOM canvas pass, producing a blob URL that becomes the `<img>` source.

## `PolyMesh` — declarative loading

The simplest way to render a mesh is `<PolyMesh src="...">`. It fetches the file, parses it, and renders a `Poly` per polygon automatically.

<Tabs syncKey="fw">
<TabItem label="React">
```tsx
import { PolyScene, PolyMesh } from "@polycss/react";

export function App() {
  return (
    <PolyScene rotX={65} rotY={45}>
      <PolyMesh src="/cottage.glb" />
    </PolyScene>
  );
}
```
</TabItem>

## Supported formats

| Format | Extension | Notes |
|--------|-----------|-------|
| OBJ + MTL | `.obj` + `.mtl` | Text format. UV maps via `vt`. Material textures from `map_Kd`. |
| glTF | `.gltf` | JSON format. Embedded or external buffers. TEXCOORD_0 UVs. |
| GLB | `.glb` | Binary glTF. Embedded textures extracted as blob URLs. |

## OBJ with MTL

When your OBJ has a companion MTL file with textures, polycss reads `map_Kd` entries and applies them as UV-mapped textures. Pass `mtlUrl` to `PolyMesh` or `useObjModel`:

```tsx
<PolyMesh
  src="/rock.obj"
  mtlUrl="/rock.mtl"
  options={{ targetSize: 40 }}
/>
```

## Material overrides

Override material colors or textures without modifying the source files:

```tsx
<PolyMesh
  src="/character.obj"
  options={{
    materialColors: { Skin: "#f4c2a1" },
    materialTextures: { Body: "/body-diffuse.png" },
    includeObjects: ["Body", "Head"],  // only these objects
  }}
/>
```

## `useMesh` hook — imperative loading

For finer control over loading state (custom spinners, error handling, etc.), use `useMesh`:

<Tabs syncKey="fw">
<TabItem label="React">
```tsx
import { PolyScene, Poly, useMesh } from "@polycss/react";

function Viewer() {
  const { polygons, loading, error, dispose } = useMesh("/cottage.glb");

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <PolyScene>
      {polygons.map((p, i) => <Poly key={i} {...p} />)}
    </PolyScene>
  );
}
```
</TabItem>
</Tabs>

## `loadMesh` — core parser (no React/Vue)

Load a mesh directly using the core parser:

```ts
import { loadMesh } from "@polycss/core";

const result = await loadMesh("/cottage.glb", { targetSize: 60 });
// result.polygons — Polygon[]
// result.dispose() — revoke blob URLs when done
```

## UV texture pipeline

For UV-textured polygons, polycss runs a one-time canvas pass per polygon at mount:

1. Extract the texture image from the file (or fetch it by URL).
2. Solve a 6-DOF affine transform from the polygon's UV coordinates to its 2D screen footprint.
3. Clip, draw, and export to a blob URL via `canvas.toBlob()`.
4. Set `<img>.src = blobUrl`.

The blob URL is revoked on unmount (call `dispose()` or let `PolyMesh` / `useMesh` handle it).

## Tips

- **`targetSize`** — scale the model so its longest axis fits this many world units (default: no scaling). Set it to roughly match the scale of your other scene content.
- **`baseUrl`** — for OBJ/glTF files with external texture paths, pass the file's URL so relative paths resolve correctly.
- For large meshes: blob URLs for embedded textures are revoked when `dispose()` is called. Always let polycss manage this — don't hold references to blob URLs across remounts.
